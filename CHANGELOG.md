# Changelog

Все заметные изменения проекта фиксируются в этом файле.


## [1.1.1] - 2026-04-09

### Added
- Добавлена проверка на активные правила от `up.sh` (интерфейс `warp` или `ip rule lookup 13335`).
- Добавлены функции `check_warp_rules_active()` и `needs_down_sh()` для определения необходимости перезапуска правил AntiZapret.
- Добавлено предупреждение в главном меню при обнаружении необходимости выполнить `down.sh && up.sh`.
- В `status_cmd` добавлен вывод состояния WARP-правил от `up.sh` и `VPN_WARP`.
- В `doctor` добавлена проверка на активные правила от `up.sh`.
- В главном меню добавлено отображение источника WARP-ключей (🔑 WARP-ключи).
- Добавлена автоматическая синхронизация ключей с `/etc/wireguard/warp.conf` при каждом запуске.
- При синхронизации ключей автоматически перезапускается `kresd@1`.

### Changed
- **Подсеть по умолчанию изменена** с `198.18.0.0/24` на `198.20.0.0/24`.
- **Приоритет WARP-ключей**: при каждом запуске проверяется `/etc/wireguard/warp.conf`. Если файл существует и содержит валидные ключи — они используются.
- При добавлении/удалении доменов с **выключенным WARPER** — выполняется только `sync_domains`, патч `kresd.conf` **не применяется**.
- Обновлены поддерживаемые ОС: Ubuntu 22.04, Ubuntu 24.04, Debian 12, Debian 13 (убран Debian 11).
- Инструкции изменены: для перезапуска правил теперь указывается `down.sh && up.sh` вместо `doall.sh`.

### Fixed
- Исправлен баг в installer: проверка `has_list_block` теперь корректно работает после полного удаления WARPER (проверяется существование файла перед grep).
- CLI-команды `add`, `remove`, `enable`, `disable` теперь корректно работают при выключенном WARPER.
- Исправлена ситуация когда после отключения `VPN_WARP` локальные ключи не работали из-за активных правил от `up.sh`.


## [1.1.0] - 2026-04-08

### Added
- **Полная совместимость с `VPN_WARP=y`**: WARPER теперь корректно работает при включённом VPN_WARP в AntiZapret.
- Автоматический поиск существующих WARP-ключей в `/etc/wireguard/warp.conf` (от VPN_WARP) при установке.
- Проверка `ANTIZAPRET_WARP=y` при установке и в runtime — если включён, выводится предупреждение и службы не запускаются.
- Новый статус `ANTIZAPRET_WARP` в выводе `warper status` и `warper doctor`.
- Предупреждение в главном меню при обнаружении конфликта с `ANTIZAPRET_WARP=y`.

### Changed
- **Патч kresd.conf теперь применяется только к `kresd@1`** (AntiZapret-клиенты), не затрагивая `kresd@2` (FullVPN-клиенты).
- Точка вставки патча изменена: теперь блок вставляется перед `-- Resolve blocked domains using Proxy Resolver`.
- Проверка в `doctor` изменена: ожидается **1 WARP-блок** вместо 2.
- Установщик теперь ищет WARP-ключи в порядке приоритета:
  1. `/etc/wireguard/warp.conf` (от VPN_WARP)
  2. `/root/warper/wgcf/wgcf-profile.conf` (существующий профиль WARPER)
  3. `/root/wgcf-profile.conf` (профиль в корне)
- При `ANTIZAPRET_WARP=y` установка продолжается, но службы не запускаются и патч не применяется.
- README полностью обновлён с описанием совместимости с VPN_WARP и новой архитектурой.

### Fixed
- Исправлен конфликт с FullVPN-клиентами при `VPN_WARP=y`: теперь они корректно проходят через встроенный WARP автора.
- Исправлена ошибка "syntax error: unexpected end of file" в warper.sh.


## [1.0.26] - 2026-04-08

### Added
- Добавлена настройка MTU sing-box через меню (Настройки → Изменить MTU).
- Добавлена настройка log level sing-box через меню (Настройки → Изменить log level).
- Доступные log level: `debug`, `info`, `warn`, `error`.
- Допустимый диапазон MTU: 1280–1500.
- MTU и log level отображаются в главном меню, `warper status` и `warper doctor`.
- При смене MTU или log level выполняется валидация конфига, перезапуск sing-box, kresd и iptables с откатом при ошибке.
- Добавлена проверка в `doctor`: ровно 1 WARP-блок в `kresd.conf`.

### Changed
- MTU по умолчанию увеличен с 1280 до 1420 (снижение фрагментации и потерь пакетов).
- `config.json.template` обновлён с новым MTU по умолчанию.
- Меню настроек расширено до 6 пунктов (добавлены log level и MTU).
- Главное меню теперь показывает текущий log level и MTU.


## [1.0.25] - 2026-04-07

### Fixed
- Исправлен баг ложного определения блоков GEMINI/CHATGPT: строка-подсказка в комментарии `domains.txt` больше не совпадает с маркером блока (заменён `grep -q` на `grep -qxF`).
- Исправлено ложное срабатывание проверки конфликта fake-подсети в `doctor`: маршрут через `singbox-tun` больше не считается конфликтом.
- Исправлен двойной патчинг `kresd.conf`: патчер стал идемпотентным — сначала удаляет старые блоки, затем вставляет ровно 1 блок в точное место.
- Исправлена установка поверх существующей: повторный запуск `install.sh` больше не раздувает `kresd.conf`.
- Пункт 4 (редактор nano) больше не предлагает применить изменения, если файл не был изменён (сравнение через каноническую форму и `sha256sum`).

### Changed
- Текст комментария-подсказки в `domains.txt` изменён: больше не содержит точный маркер блока.
- Пользовательские домены теперь добавляются под заголовок `# Пользовательские домены:`, а не в конец файла.
- Добавлена функция `rebuild_master_file()` для нормализации структуры `domains.txt` (заголовок → пользовательские домены → блоки GEMINI/CHATGPT).
- Добавлена функция `normalize_include_ips()` для удаления пустых строк и дубликатов в `include-ips.txt`.
- `install.sh`, `warper.sh` и `uninstaller.sh` вызывают `normalize_include_ips()` после каждого изменения `include-ips.txt`.
- Проверка наличия блоков GEMINI/CHATGPT унифицирована через функцию `has_list_block()` с `grep -qxF`.
- Проверка конфликта fake-подсети (`subnet_conflicts()`) теперь корректно игнорирует маршруты и адреса на интерфейсе `singbox-tun`.


## [1.0.24] - 2026-06-04

### Added
- Добавлена команда `warper status` для краткого статуса системы.
- Добавлены CLI-команды без меню:
  - `warper add DOMAIN`
  - `warper remove DOMAIN`
  - `warper enable gemini|chatgpt`
  - `warper disable gemini|chatgpt`
  - `warper sync`
  - `warper patch`
  - `warper doctor`
  - `warper status`
- Добавлена безопасная работа с `config.json` через `jq`.
- Добавлена проверка конфликта fake-подсети с локальными интерфейсами, маршрутами и Docker-сетями.
- Расширен `doctor`: теперь он проверяет больше системных параметров и права на чувствительные файлы.

### Changed
- `install.sh` теперь устанавливает `jq` как зависимость.
- Извлечение WARP-данных из `config.json` переведено на `jq`, с fallback на старый способ.
- README обновлён под новые команды и диагностику.
- Улучшена обработка Docker-проверок при поиске конфликтов fake-подсети.
- Подчищены shell/runtime шероховатости в CLI-командах и диагностике.
- Улучшены проверки прав и поведение в пограничных сценариях.


## [1.0.23] - 2026-04-04

### Added
- Добавлена команда `warper doctor` для диагностики состояния установки.
- Добавлено создание резервной копии `kresd.conf` перед первым патчем.
- Добавлено восстановление `kresd.conf` из backup при удалении патча и при деинсталляции.
- Добавлена безопасная загрузка файлов при обновлении утилиты.
- Добавлена проверка валидности доменов в скачанных списках перед импортом.
- Добавлена проверка списков доменов на дубликаты и мусор до записи в рабочие файлы.
- Добавлен GitHub Actions workflow с `shellcheck`.

### Fixed
- Исправлен баг смены fake-подсети в `settings_menu`: старая подсеть теперь корректно удаляется из `include-ips.txt`.
- Улучшена логика патчинга `kresd.conf`: теперь проверяется наличие точек вставки, а при их отсутствии возвращается ошибка.
- Добавлена проверка успешного запуска `sing-box` после `start/restart`.
- Добавлена валидация конфигурации через `sing-box check -c /etc/sing-box/config.json`.

### Changed
- Исключено дублирование правил `iptables` с помощью `iptables -C`.
- Unit-файлы `systemd` теперь устанавливаются в `/etc/systemd/system`.
- Чтение `warper.conf` переведено на безопасный парсинг без `source`.
- README полностью обновлён и приведён к актуальному поведению проекта.
