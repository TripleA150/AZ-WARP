## [1.2.1] - 2026-04-28

### Added
- **Режим WG (WireGuard-соединение)**: новый тип маршрутизации — трафик доменов направляется через собственный WireGuard-сервер.
  - Поддерживается любой стандартный `.conf` файл WireGuard.
  - Автоматический поиск `.conf` файлов в `/root/` и `/root/warper/`.
  - Автоматическое исключение файлов Cloudflare WARP (wgcf-profile.conf, warp.conf) из списка WG-конфигов.
  - Ввод параметров вручную (Endpoint, Address, PrivateKey, PublicKey, PresharedKey, PersistentKeepalive).
  - Сохранение последнего WG-подключения — при повторном выборе предлагается использовать сохранённые данные.
  - Отображение endpoint WG-соединения в главном меню и `warper status`.
- Новый шаблон конфигурации: `templates/config-wg.json.template`.
- При установке (`install.sh`) добавлен выбор режима **WG** в дополнение к WARP и Slave.
- Интерактивный выбор WG-конфига при переключении режима в `warper`.
- В настройках (`warper` → Настройки → Режим маршрутизации) добавлен пункт **3 — WG**.
- **Управление WARP-ключами** (`warper` → Настройки → 8):
  - Выбор источника ключей: системный `/etc/wireguard/warp.conf`, локальный профиль wgcf, генерация новых.
  - Приоритетное отображение источника ключей с исключением невалидных файлов (не от Cloudflare).
- При установке в режиме WARP добавлен интерактивный выбор источника ключей вместо автоматического захвата первого найденного.
- Исправлено определение источника WARP-ключей в главном меню: применяется корректная приоритетная логика (system → wgcf → root), без перезаписи ранее найденного источника.

### Changed
- **Фильтрация WG-конфигов**: файлы Cloudflare WARP (по публичному ключу `bmXOC+...`) теперь исключаются при поиске WG-конфигов и при отображении источников WARP-ключей.
- **PresharedKey обязателен** для режима WG — конфиги без него отклоняются при парсинге.
- При установке в режиме WARP теперь всегда показывается меню выбора источника ключей (вместо молчаливого захвата первого найденного).
- Генерация WARP-ключей в установщике переработана: убраны `if !` проверки для `wgcf register/generate` (используется `|| true`), реальная ошибка определяется по отсутствию `wgcf-profile.conf`.
- Файлы шаблонов перенесены в директорию `templates/` (обратная совместимость: копии оставлены в корне репозитория для обновления со старых версий).
- `rebuild_config()` теперь учитывает режим `wg` и вызывает `rebuild_config_wg()`.
- `check_and_sync_warp_keys()` пропускает синхронизацию в режимах `slave` и `wg`.
- Логика определения `WARP_KEYS_SRC` в `show_main_menu()` исправлена: используется `else-if` цепочка, источник с более высоким приоритетом не перезаписывается.

### Fixed
- Исправлена ошибка `unbound variable: warp_creds_info` при переключении на режим WARP.
- Исправлено некорректное определение источника WARP-ключей в главном меню (невалидные wgcf-profile.conf от WG-соединений больше не показываются как источник).
- Исправлен баг установщика: при выборе "Сгенерировать новый ключ WARP" скрипт завершался с ошибкой из-за ненулевого кода возврата `wgcf register`.
- Исправлены ошибки `local: can only be used in a function` — убраны `local` объявления из тела скрипта в `install.sh`.
- Исправлены ошибки `unbound variable` в `warper.sh` при `set -uo pipefail` — все `local` переменные инициализируются при объявлении.
- Исправлена невозможность ввода адреса slave-сервера при установке (буфер stdin).


## [1.2.0] - 2026-04-26

### Added
- **Режим Slave (донор-сервер)**: трафик направляется через внешний сервер по Shadowsocks-каналу.
- **WARPERSLAVE** — отдельный компонент для донор-сервера.
  - `install-slave.sh` — установщик.
  - `warperslave.sh` — утилита управления (меню + CLI).
  - `uninstall-slave.sh` — деинсталлятор.
  - `sing-box-slave.service` — отдельный systemd unit.
  - `versionslave` — файл версии для проверки обновлений.
- Два режима донор-сервера: **Direct** и **WARP**.
- Переключение режима маршрутизации WARP ↔ Slave в `warper`.
- Выбор режима при установке (WARP или Slave).
- Сохранение данных slave-подключения — при возврате предлагается использовать сохранённое.
- В `warper status`: режим маршрутизации, адрес slave-сервера, ключ Shadowsocks.
- Показ начала SS-ключа в главном меню, команда для полного вывода.
- Система обновлений warperslave: `warperslave update`.
- Настройки warperslave: log level и MTU для режима WARP.
- Lock-файлы для предотвращения параллельного запуска.
- Функция `is_interactive()` — CLI-команды не блокируют stdin.
- Локальное определение IPv4 через `ip -4 route get 1.1.1.1`.
- Валидация ответа версии — `404: Not Found` больше не попадает в меню.

### Changed
- `set -uo pipefail` добавлен в `warper.sh`.
- Slave-конфиги sing-box: `dns` с `strategy: "ipv4_only"`.
- `get_remote_version()`: `curl -4 -sf` + regex-валидация.
- `subnet_conflicts()`: Docker IDs через `mapfile`.
- `check_and_sync_warp_keys()` не блокирует CLI-режим.
- Полный перезапуск sing-box (`stop/start`) при смене режима.
- Требования к ОС для WARPERSLAVE: Ubuntu 20.04+, Debian 10+.
- README и документация разделены: основная информация в README, детали в `docs/`.

### Fixed
- `docker network inspect $ids` — word splitting при множестве сетей.
- `check_and_sync_warp_keys()` блокировал CLI-режим.
- `404: Not Found` в меню при недоступности версии.
- Утечка IPv6 в конфигурациях WARPERSLAVE.
- Первый переход Slave → WARP без ручного перезапуска sing-box.
