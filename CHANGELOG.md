# Changelog

Все заметные изменения проекта фиксируются в этом файле.

## [1.3.5] - 2026-06-02

### Fixed

- **MTU не менялся в режиме Slave**: `set_mtu` пытался изменить значение в `.endpoints[0].mtu`, но в Slave-конфиге `endpoints` отсутствует (Shadowsocks — не туннельный outbound). Теперь функция определяет тип конфига и меняет MTU в правильном месте — на `endpoints` для WARP/WG или на `inbounds[type=tun]` для Slave.
- В шаблоны конфигов добавлен дефолтный `mtu: 1420` на TUN-интерфейс — предотвращает использование значения по умолчанию sing-box (9000) которое может вызывать фрагментацию при использовании Shadowsocks.

## [1.3.4] - 2026-06-02

### Fixed

- **DNS-резолвинг доменных имён в outbound/endpoint sing-box**: исправлена ошибка при которой sing-box не мог корректно резолвить домены endpoint'а WireGuard / Shadowsocks-сервера потому что использовал DNS-сервер через ещё не поднятый туннель. Теперь резолвинг идёт через локальный публичный DNS (8.8.8.8) напрямую с сервера.

  Особенно актуально для:
  - WARP с endpoint в виде домена (например `engage.cloudflareclient.com:2408` вместо IP)
  - WG-режима если в `.conf` файле указан Endpoint доменом (например `vpn.example.com:51820`)
  - Slave-режима если адрес донор-сервера задан доменным именем

  Благодарность [@TripleA150](https://github.com/TripleA150) за обнаружение проблемы и предложенный фикс.

### Changed

- Шаблоны sing-box обновлены — добавлен дополнительный DNS-сервер `local` (UDP 8.8.8.8), `default_domain_resolver` переключён на него:
  - `templates/config.json.template` (WARP)
  - `templates/config-slave-master.json.template` (Slave)
  - `templates/config-wg.json.template` (WG)
- **Умная пересборка config.json при обновлении**: `warper update` теперь сравнивает хэши шаблонов до и после обновления. Если шаблон активного режима изменился — конфиг пересобирается автоматически и sing-box перезапускается. Если шаблон не менялся — sing-box не трогается, обновление происходит быстрее и без прерывания работы.

## [1.3.3] - 2026-06-01

### Added

- **🌐 Веб-панель управления WARPER** — браузерный интерфейс на Flask + HTMX (опциональная установка).
  - Полное управление через UI: домены, IP-подсети, sing-box, режимы маршрутизации (WARP/Slave/WG), все настройки.
  - Просмотр логов sing-box в реальном времени с фильтром по уровню и автообновлением.
  - Запуск warper doctor из браузера.
  - Редактор `domains.txt` и `ip-ranges.txt` с сохранением пользовательских комментариев.
  - Загрузка WG-конфигов через браузер (drag & drop).
  - Управление WARP-ключами: выбор источника, генерация новых.
  - Проверка обновлений WARPER на dashboard с одной кнопкой "Обновить" (показывает прогресс в реальном времени через Server-Sent Events).
  - Адаптивный тёмно-зелёный дизайн, мобильный и десктоп.
  - Офлайн-зависимости (Tailwind CSS, HTMX встроены — не зависят от CDN при работе).
- **Безопасность веб-панели**:
  - bcrypt-хеши паролей в `web/data/users.json` (chmod 600, доступ только root).
  - SECRET_KEY Flask в `web/data/secret.key` (генерируется автоматически), ротируется при смене пароля → все активные сессии становятся невалидными.
  - Защита от brute-force: 10 попыток / 10 минут → блокировка IP на 15 минут (persistent на диске).
  - Timing-safe проверка пароля (всегда выполняется bcrypt с одинаковым cost).
  - CSRF-защита через проверку Origin/Referer.
  - X-Real-IP только от nginx — невозможно подделать заголовком.
  - Security-заголовки nginx: X-Frame-Options, X-Content-Type-Options, HSTS (для HTTPS).
  - Аудит-лог авторизации в `web/data/auth.log` с автоматической ротацией.
  - Cookie: HttpOnly, SameSite=Lax, Secure при HTTPS.
- **Установщик веб-панели** (`install-web.sh`):
  - Выбор порта с проверкой занятости.
  - HTTPS опционально: Let's Encrypt с доменом или самоподписанный сертификат.
  - Скрытый ввод пароля с подтверждением или автогенерация безопасного пароля.
  - Интеграция в основной `install.sh` (опциональный запрос установки веб-панели).
- **CLI-команды для управления веб-панелью**:
  - `warper webpass` — интерактивная смена логина/пароля.
  - `warper webpass USER PASS` — неинтерактивная смена.
  - `warper webpass --reset` — полный сброс, создаёт admin со случайным паролем.
  - `warper webpass --unblock` — снять блокировки IP.
  - `warper webupdate` — обновление файлов веб-панели.
- **Меню веб-панели в `warper` (пункт `W`)**:
  - Установка/удаление веб-панели прямо из главного меню.
  - Смена логина/пароля, сброс блокировок IP, смена порта.
  - Просмотр логов сервиса и лога авторизаций.
  - Управление сервисом (старт/стоп/рестарт).
- **Новые CLI-команды для автоматизации и веб-панели**:
  - `warper toggle` — включить/выключить WARPER.
  - `warper mode warp|slave|wg [args]` — смена режима маршрутизации из CLI.
  - `warper subnet X.X.X.0/M` — изменение fake-подсети.
  - `warper loglevel debug|info|warn|error` — log level sing-box.
  - `warper mtu N` — MTU sing-box.
  - `warper autopatch on|off` — управление автопатчем DNS.
  - `warper fullvpn on|off` — FullVPN WARP-резолвинг.
  - `warper iproutemode antizapret|all_vpn|all` — режим применения IP-маршрутов.
  - `warper ipexport on|off` — экспорт CIDR в AntiZapret.
  - `warper warpkey list|generate` — управление WARP-ключами.
  - `warper wgconfig list` — список доступных WG-конфигов.
  - `warper status json` — статус в JSON-формате (используется веб-панелью).
  - `warper logs [N]` — последние N строк логов sing-box.
- **Единое обновление**: команда `warper update` теперь обновляет и WARPER, и веб-панель (если установлена). Запускается из CLI, главного меню или из браузера.
- **Сохранение пользовательских комментариев**:
  - `domains.txt`: комментарии (`#`) и пустые строки в пользовательском блоке сохраняются при `warper add/remove`, переключении списков Gemini/ChatGPT, синхронизации.
  - `ip-ranges.txt`: комментарии и пустые строки сохраняются при `warper ipadd/ipremove`.

### Changed

- `uninstaller.sh` теперь автоматически удаляет веб-панель (сервис, nginx-конфиг, файлы) перед удалением WARPER.
- `rebuild_master_file()` переписан для сохранения пользовательских комментариев в `domains.txt`.
- `insert_user_domain()` вставляет домен без пересборки файла — сохраняет форматирование и комментарии.
- `enable_disable_list()` для Gemini/ChatGPT работает напрямую с файлом, не пересобирая пользовательский блок.
- `cli_remove_domain()` удаляет только строку с доменом через `sed`, не трогая комментарии.
- `add_ip_range()` / `remove_ip_range()` добавляют/удаляют только нужные строки в `ip-ranges.txt`.
- Lock-файл WARPER: блокировка применяется только для тяжёлых команд (`toggle`, `sync`, `ipsync`, `mode`, `subnet`, `patch`, `update`). Read-only команды и TUI-меню работают без lock — веб-панель может вызывать `warper` параллельно с открытым TUI в SSH.

### Fixed

- Устранены проблемы с lock-файлом.

## [1.3.0] - 2026-04-30

### Added
- **Маршрутизация по IP-подсетям (CIDR)**: новая функция — направление трафика к конкретным IPv4-подсетям через sing-box tun.
  - Файл `/root/warper/ip-ranges.txt` для хранения подсетей в формате CIDR (A.B.C.D/M).
  - Интерактивное меню управления: добавление, удаление, редактирование (nano), синхронизация.
  - Три режима применения IP-маршрутов:
    - **Только AntiZapret** — маршруты действуют только для AntiZapret-клиентов (policy routing через `ip rule` + `table 100`).
    - **AntiZapret + FullVPN** — маршруты действуют для обоих типов VPN-клиентов (для для VPN_WARP=y в том числе).
    - **Весь трафик сервера (Beta)** — маршруты в main table + table 13335.
  - Автоматическое определение клиентских подсетей из `/root/antizapret/setup` (поддержка `ALTERNATIVE_CLIENT_IP`, `CLIENT_IP`).
  - Двусторонняя синхронизация: файл `ip-ranges.txt` ↔ kernel routes ↔ `ip-ranges.applied`.
  - Автоматическое добавление CIDR в ipset `antizapret-forward` при `RESTRICT_FORWARD=y` — трафик не блокируется фаерволом AntiZapret.
  - **Экспорт в AntiZapret**: опциональная запись CIDR в `/root/antizapret/config/warper-include-ips.txt` + автозапуск `doall.sh ip` — AntiZapret-клиенты начинают маршрутизировать эти сети через VPN.
  - Экспорт в AntiZapret включён по умолчанию.
  - Автоматическая пересинхронизация IP-маршрутов после перезапуска sing-box (restart, start, смена подсети, смена MTU/log level).
  - CLI-команды: `warper ipadd`, `warper ipremove`, `warper ipsync`, `warper iplist`, `warper iproutes`.
  - Статус IP-маршрутов в `warper status`, `warper doctor` и главном меню.
  - Валидация CIDR с защитой от loopback, link-local, multicast и fake-подсети WARPER.
  - Очистка IP-маршрутов, ip rule и `warper-include-ips.txt` при деинсталляции WARPER.
- **Безопасное обновление (bulletproof update)**: файлы сначала скачиваются во временную директорию, проверяются и только потом заменяют рабочие.
  - Проверка синтаксиса bash-скриптов (`bash -n`) при обновлении.
  - Проверка шаблонов конфигурации и systemd unit-файлов перед установкой.
  - Полный backup текущих файлов перед обновлением.
  - Автоматический откат при неудачном обновлении (ошибка установки, невалидный конфиг, sing-box не запустился).
- Обновлён `warper-autopatch.service`: при загрузке системы выполняется `warper patch` + `warper ipsync`.

### Changed
- Режим "Весь трафик сервера" помечен как Beta — для FullVPN-клиентов с `VPN_WARP=y` маршруты добавляются также в table 13335.
- `sync_ip_ranges()` сравнивает желаемое состояние с реальными kernel routes — корректно восстанавливает маршруты после перезапуска sing-box.
- При деинсталляции WARPER: `doall.sh ip` вызывается один раз в конце (вместо двух раз).
- При деинсталляции с сохранением настроек: `ip-ranges.txt` сохраняется вместе с `domains.txt` и `warper.conf`.
- Счётчики маршрутов используют `((var+=1))` вместо `((var++))` — устранены ложные ошибки при `set -u`.
- Временные файлы во всех функциях очищаются явным `rm -f` вместо `trap ... RETURN` — устранены ошибки `unbound variable` при `set -uo pipefail`.

### Fixed
- Устранена ошибка `unbound variable: desired_tmp` при выходе из меню и при отображении главного экрана.
- Устранена ошибка `local: can only be used in a function` в `warperslave.sh`.
- Устранена ошибка `unbound variable: WGCF_DIR` в `install-slave.sh`.
- Устранён word splitting в `select_wg_config()` при именах файлов с пробелами.
- При смене режима IP-маршрутизации stale routes из предыдущего режима корректно удаляются.
- Удалена дублирующаяся функция `detect_client_subnets()`.

## [1.2.1] - 2026-04-28

### Added
- **Режим WG (WireGuard-соединение)**: новый тип маршрутизации — трафик доменов направляется через собственный WireGuard-сервер.
  - Поддерживается любой стандартный `.conf` файл WireGuard.
  - Автоматический поиск `.conf` файлов в `/root/` и `/root/warper/`.
  - Автоматическое исключение файлов Cloudflare WARP (wgcf-profile.conf, warp.conf) из списка WG-конфигов.
  - Ввод параметров вручную (Endpoint, Address, PrivateKey, PublicKey, PresharedKey, PersistentKeepalive).
  - Сохранение последнего WG-подключения — при повторном выборе предлагается использовать сохранённые данные.
  - Отображение endpoint WG-соединения в главном меню и `warper status`.
- Новый шаблон конфигурации: `templates/config-wg.json.template`.
- При установке (`install.sh`) добавлен выбор режима **WG** в дополнение к WARP и Slave.
- Интерактивный выбор WG-конфига при переключении режима в `warper`.
- В настройках (`warper` → Настройки → Режим маршрутизации) добавлен пункт **3 — WG**.
- **Управление WARP-ключами** (`warper` → Настройки → 8):
  - Выбор источника ключей: системный `/etc/wireguard/warp.conf`, локальный профиль wgcf, генерация новых.
  - Приоритетное отображение источника ключей с исключением невалидных файлов (не от Cloudflare).

  - `install-slave.sh` — установщик.
  - `warperslave.sh` — утилита управления (меню + CLI).
  - `uninstall-slave.sh` — деинсталлятор.
  - `sing-box-slave.service` — отдельный systemd unit.
  - `versionslave` — файл версии для проверки обновлений.
- Два режима донор-сервера: **Direct** и **WARP**.
- Переключение режима маршрутизации WARP ↔ Slave в `warper`.
- Выбор режима при установке (WARP или Slave).
- Сохранение данных slave-подключения — при возврате предлагается использовать сохранённое.
- В `warper status`: режим маршрутизации, адрес slave-сервера, ключ Shadowsocks.
- Показ начала SS-ключа в главном меню, команда для полного вывода.
- Система обновлений warperslave: `warperslave update`.
- Настройки warperslave: log level и MTU для режима WARP.
- Lock-файлы для предотвращения параллельного запуска.
- Функция `is_interactive()` — CLI-команды не блокируют stdin.
- Локальное определение IPv4 через `ip -4 route get 1.1.1.1`.
- Валидация ответа версии — `404: Not Found` больше не попадает в меню.

### Changed
- `set -uo pipefail` добавлен в `warper.sh`.
- Slave-конфиги sing-box: `dns` с `strategy: "ipv4_only"`.
- `get_remote_version()`: `curl -4 -sf` + regex-валидация.

- README и документация разделены: основная информация в README, детали в `docs/`.

### Fixed
